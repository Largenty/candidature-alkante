Étape 01 — Le cap · 48.135° N, 1.535° O

Pourquoi mettre le cap sur Alkante

Le web que je préfère est celui qui sert à quelque chose de tangible. Vos projets manipulent le territoire, l'environnement, l'espace public : des données qui décrivent le réel, et des outils que des gens utilisent vraiment.

La géomatique

Croiser du code et des données spatiales, c'est donner une forme visible à des enjeux concrets. C'est exactement le type de défi technique qui me fait ouvrir mon éditeur le matin avec envie.

La diversité des projets

Des ministères aux grands groupes, du SI au mobile : une équipe de 60 personnes qui couvre ce spectre, c'est la garantie de ne jamais s'installer dans une routine — et de progresser en continu.

La culture d'équipe

Autonomie, organisation peu hiérarchique, équilibre vie pro / vie perso : c'est le cadre dans lequel je travaille déjà en indépendant. Ce qui me manque, c'est une équipe avec qui le partager dans la durée.

Étape 02 — Le relevé

Relevé de compétences, point par point

Votre annonce à gauche, mon terrain à droite. Symboles conformes à la légende.

En production / au quotidien Fondations solides, montée en puissance rapide

Front-end · React

Mon outil quotidien — et ma matière d'enseignement. Je forme des développeurs à React sur un cursus complet, du premier hook aux patterns avancés. Enseigner une technologie oblige à la comprendre en profondeur.

Back-end · Node.js, Express

Express en production et en salle de cours : API REST, authentification JWT, architecture MVC, bonnes pratiques de structuration. J'ai bâti des supports de formation complets sur ces sujets.

Bases de données · PostgreSQL, MongoDB

PostgreSQL en production via Supabase et Prisma ; MongoDB et Mongoose sur mes projets back-end récents. Modélisation, migrations, optimisation de requêtes : le quotidien d'un full-stack.

Outils · Git, Docker, Jira, Figma, Postman

L'outillage standard de mes journées. Figma en particulier : formé au design UX/UI, mes maquettes précèdent toujours mon code — un vrai atout pour dialoguer avec les utilisateurs comme avec les designers.

Qualité · Tests, CI, documentation

Des habitudes que je transmets à mes étudiants — donc que je m'impose d'abord à moi-même. Revue de code, documentation claire, amélioration continue : la pédagogie rend rigoureux.

Symfony / PHP

Parlons franchement : mon quotidien est côté JavaScript. Mais MVC, ORM, injection de dépendances, POO, API REST — les fondations sur lesquelles Symfony est construit sont précisément celles que je pratique et que j'enseigne. Un framework s'apprend en quelques semaines ; l'architecture, elle, est déjà là.

Pédagogie · Relationnel

Formateur web en poste : vulgariser, documenter, accompagner des débutants comme des utilisateurs finaux, c'est littéralement mon métier. « Aisance relationnelle et pédagogie » n'est pas une ligne de CV, c'est ma fiche de poste actuelle.
